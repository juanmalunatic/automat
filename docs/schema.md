# Schema

This document is the database source of truth for the data-complete staged MVP.

SQLite is the MVP database.

Timestamps should be stored as ISO-8601 text in UTC when generated by the system, unless a source field is explicitly user-local or source-local.

JSON fields are stored as TEXT containing valid JSON.

## 1. `ingestion_runs`

One row per polling/import run.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `started_at TEXT NOT NULL`
- `finished_at TEXT`
- `source_name TEXT NOT NULL`
- `query_config_json TEXT`
- `status TEXT NOT NULL`
- `error_message TEXT`
- `jobs_fetched_count INTEGER DEFAULT 0`
- `jobs_new_count INTEGER DEFAULT 0`
- `jobs_updated_count INTEGER DEFAULT 0`

Suggested statuses:

- `running`
- `success`
- `failed`

## 2. `raw_job_snapshots`

Raw Upwork/API/scrape payloads.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `ingestion_run_id INTEGER REFERENCES ingestion_runs(id)`
- `upwork_job_id TEXT`
- `fetched_at TEXT NOT NULL`
- `source_query TEXT`
- `raw_json TEXT NOT NULL`
- `raw_hash TEXT`

Indexes:

- `idx_raw_job_snapshots_job_id`
- `idx_raw_job_snapshots_run_id`

## 3. `job_snapshots_normalized`

Normalized visible data from a raw job snapshot.

This table stores the original decision surface before AI/economics/final verdict.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `raw_snapshot_id INTEGER REFERENCES raw_job_snapshots(id)`
- `upwork_job_id TEXT`
- `normalized_at TEXT NOT NULL`
- `normalizer_version TEXT NOT NULL`

Meta/source:

- `id_original TEXT`
- `action TEXT`
- `time_action TEXT`
- `source_url TEXT`

Client history:

- `c_verified_payment INTEGER`
- `c_verified_phone INTEGER`
- `c_country TEXT`
- `c_hist_jobs_posted INTEGER`
- `c_hist_jobs_open INTEGER`
- `c_hist_hire_rate REAL`
- `c_hist_total_spent REAL`
- `c_hist_hires_total INTEGER`
- `c_hist_hires_active INTEGER`
- `c_hist_avg_hourly_rate REAL`
- `c_hist_hours_hired REAL`
- `c_hist_member_since TEXT`

Job core:

- `j_title TEXT`
- `j_description TEXT`
- `j_mins_since_posted INTEGER`
- `j_posted_at TEXT`
- `j_apply_cost_connects INTEGER`
- `j_project_type TEXT`
- `j_contract_type TEXT`
- `j_pay_fixed REAL`
- `j_pay_hourly_low REAL`
- `j_pay_hourly_high REAL`
- `j_skills TEXT`
- `j_qualifications TEXT`

Activity/competition:

- `a_proposals TEXT`
- `a_mins_since_cli_viewed INTEGER`
- `a_hires INTEGER`
- `a_interviewing INTEGER`
- `a_invites_sent INTEGER`
- `a_invites_unanswered INTEGER`

Market sanity:

- `mkt_high REAL`
- `mkt_avg REAL`
- `mkt_low REAL`

Status metadata:

- `field_status_json TEXT`
- `created_at TEXT NOT NULL`

Constraints:

- `j_contract_type` should be one of `fixed`, `hourly`, `NOT_VISIBLE`, `PARSE_FAILURE`, or NULL.
- zero is different from NULL.
- unavailable values should use NULL plus `field_status_json`.

Indexes:

- `idx_job_snapshots_normalized_job_id`
- `idx_job_snapshots_normalized_created_at`

## 4. `triage_settings_versions`

Versioned economic assumptions.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `created_at TEXT NOT NULL`
- `name TEXT NOT NULL UNIQUE`
- `target_rate_usd REAL NOT NULL`
- `low_cash_mode INTEGER NOT NULL`
- `connect_cost_usd REAL NOT NULL`
- `p_strong REAL NOT NULL`
- `p_ok REAL NOT NULL`
- `p_weak REAL NOT NULL`
- `fbv_hours_defined_short_term REAL NOT NULL`
- `fbv_hours_ongoing_or_vague REAL NOT NULL`
- `is_default INTEGER NOT NULL DEFAULT 0`

Default MVP row:

- `name = default_low_cash_v1`
- `target_rate_usd = 25`
- `low_cash_mode = 1`
- `connect_cost_usd = 0.15`
- `p_strong = 0.01400`
- `p_ok = 0.00189`
- `p_weak = 0.00020`
- `fbv_hours_defined_short_term = 10`
- `fbv_hours_ongoing_or_vague = 8`
- `is_default = 1`

## 5. `filter_results`

Deterministic pre-AI filtering and scoring.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `job_snapshot_id INTEGER NOT NULL REFERENCES job_snapshots_normalized(id)`
- `filter_version TEXT NOT NULL`
- `created_at TEXT NOT NULL`
- `passed INTEGER NOT NULL`
- `routing_bucket TEXT NOT NULL`
- `score REAL`
- `reject_reasons_json TEXT`
- `positive_flags_json TEXT`
- `negative_flags_json TEXT`

Allowed routing buckets:

- `DISCARD`
- `LOW_PRIORITY_REVIEW`
- `MANUAL_EXCEPTION`
- `AI_EVAL`

Indexes:

- `idx_filter_results_job_snapshot_id`

## 6. `ai_evaluations`

AI semantic judgment.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `job_snapshot_id INTEGER NOT NULL REFERENCES job_snapshots_normalized(id)`
- `settings_version_id INTEGER REFERENCES triage_settings_versions(id)`
- `model TEXT`
- `prompt_version TEXT NOT NULL`
- `created_at TEXT NOT NULL`
- `input_json TEXT`
- `output_json TEXT`

Qualitative fields:

- `ai_quality_client TEXT`
- `ai_quality_fit TEXT`
- `ai_quality_scope TEXT`
- `ai_price_scope_align TEXT`
- `ai_verdict_bucket TEXT`
- `ai_likely_duration TEXT`

Risk/control fields:

- `proposal_can_be_written_quickly INTEGER`
- `scope_explosion_risk INTEGER`
- `severe_hidden_risk INTEGER`

Reason/evidence fields:

- `ai_reason_apply_short TEXT`
- `ai_best_reason_to_apply TEXT`
- `ai_why_trap TEXT`
- `ai_proposal_angle TEXT`
- `fit_evidence_json TEXT`
- `client_evidence_json TEXT`
- `scope_evidence_json TEXT`
- `risk_flags_json TEXT`

Allowed qualitative values:

- `ai_quality_client`: `Strong`, `Ok`, `Weak`
- `ai_quality_fit`: `Strong`, `Ok`, `Weak`
- `ai_quality_scope`: `Strong`, `Ok`, `Weak`
- `ai_price_scope_align`: `aligned`, `underposted`, `overpriced`, `unclear`
- `ai_verdict_bucket`: `Strong`, `Ok`, `Weak`, `No`
- `ai_likely_duration`: `defined_short_term`, `ongoing_or_vague`

Indexes:

- `idx_ai_evaluations_job_snapshot_id`

## 7. `economics_results`

Deterministic apply-stage economics.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `job_snapshot_id INTEGER NOT NULL REFERENCES job_snapshots_normalized(id)`
- `settings_version_id INTEGER NOT NULL REFERENCES triage_settings_versions(id)`
- `ai_evaluation_id INTEGER REFERENCES ai_evaluations(id)`
- `created_at TEXT NOT NULL`
- `economics_version TEXT NOT NULL`

Economic fields:

- `j_apply_cost_connects INTEGER`
- `b_apply_cost_usd REAL`
- `b_apply_prob REAL`
- `b_first_believ_value_usd REAL`
- `b_required_apply_prob REAL`
- `b_calc_max_rac_usd REAL`
- `b_margin_usd REAL`
- `b_calc_max_rac_connects INTEGER`
- `b_margin_connects INTEGER`

Status fields:

- `calc_status TEXT NOT NULL`
- `calc_error TEXT`

Suggested `calc_status` values:

- `ok`
- `parse_failure`
- `missing_prerequisite`
- `not_applicable`

Indexes:

- `idx_economics_results_job_snapshot_id`

## 8. `triage_results`

Final decision after filter, AI, and economics.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `job_snapshot_id INTEGER NOT NULL REFERENCES job_snapshots_normalized(id)`
- `settings_version_id INTEGER REFERENCES triage_settings_versions(id)`
- `filter_result_id INTEGER REFERENCES filter_results(id)`
- `ai_evaluation_id INTEGER REFERENCES ai_evaluations(id)`
- `economics_result_id INTEGER REFERENCES economics_results(id)`
- `created_at TEXT NOT NULL`
- `triage_version TEXT NOT NULL`

Final fields:

- `ai_verdict_apply TEXT`
- `ai_apply_promote TEXT`
- `final_verdict TEXT NOT NULL`
- `queue_bucket TEXT NOT NULL`
- `priority_score REAL`
- `final_reason TEXT`

Allowed final verdicts:

- `APPLY`
- `MAYBE`
- `NO`

Allowed promotion traces:

- `none`
- `ok_override_to_maybe`
- `ok_override_to_apply`
- `low_cash_maybe_to_apply`

Suggested queue buckets:

- `HOT`
- `REVIEW`
- `MANUAL_EXCEPTION`
- `ARCHIVE`

Indexes:

- `idx_triage_results_job_snapshot_id`

## 9. `user_actions`

Tracks what the user actually did.

Fields:

- `id INTEGER PRIMARY KEY AUTOINCREMENT`
- `upwork_job_id TEXT`
- `job_snapshot_id INTEGER REFERENCES job_snapshots_normalized(id)`
- `created_at TEXT NOT NULL`
- `action TEXT NOT NULL`
- `notes TEXT`

Suggested actions:

- `seen`
- `applied`
- `skipped`
- `saved`
- `bad_recommendation`
- `good_recommendation`
- `client_replied`
- `interview`
- `hired`

Indexes:

- `idx_user_actions_upwork_job_id`
- `idx_user_actions_job_snapshot_id`

Main user-facing view.

The view should select the latest triage result for each `upwork_job_id` and join:

- `job_snapshots_normalized`
- `filter_results`
- `ai_evaluations`
- `economics_results`
- `triage_results`

It should include:

- `queue_bucket`
- `final_verdict`
- `priority_score`
- `ai_verdict_bucket`
- `ai_verdict_apply`
- `ai_apply_promote`
- `ai_quality_fit`
- `ai_quality_client`
- `ai_quality_scope`
- `ai_price_scope_align`
- `ai_likely_duration`
- `b_margin_usd`
- `b_required_apply_prob`
- `b_first_believ_value_usd`
- `b_apply_cost_usd`
- `j_apply_cost_connects`
- `j_title`
- `source_url`
- `j_mins_since_posted`
- `j_contract_type`
- `j_pay_fixed`
- `j_pay_hourly_low`
- `j_pay_hourly_high`
- `c_verified_payment`
- `c_country`
- `c_hist_total_spent`
- `c_hist_hire_rate`
- `c_hist_avg_hourly_rate`
- `a_proposals`
- `a_interviewing`
- `a_invites_sent`
- `a_mins_since_cli_viewed`
- `ai_reason_apply_short`
- `ai_best_reason_to_apply`
- `ai_why_trap`
- `ai_proposal_angle`
- `fit_evidence_json`
- `client_evidence_json`
- `scope_evidence_json`

- `reject_reasons_json`
- `positive_flags_json`
- `negative_flags_json`
